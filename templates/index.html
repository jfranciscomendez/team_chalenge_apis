📘 Documentación de la API

/inicio

Descripción: Landing page que explica los distintos endpoints disponibles en esta API y cómo usarlos. Es ideal para usuarios que visitan la aplicación por primera vez.

Método: GET

Contenido: HTML informativo y accesible con enlaces y descripciones de los demás endpoints.

/features

Descripción: Este endpoint describe las características (features) necesarias para realizar una predicción.

Método: GET

Ejemplo de respuesta:

{
  "Ram_GB": "Cantidad de memoria RAM en gigabytes",
  "Weight_kg": "Peso del portátil en kilogramos",
  "Inches": "Tamaño de la pantalla en pulgadas"
}

/api/v1/predict

Descripción: Este endpoint permite hacer predicciones utilizando un modelo de machine learning.

Método: GET

Parámetros esperados (como query en la URL):

  • ram_gb: cantidad de memoria RAM (ej: 8, 16)
  • weight_kg: peso del dispositivo en kilogramos (ej: 2.3)
  • inches: tamaño de pantalla en pulgadas (ej: 15.6)

Ejemplo de URL:

http://localhost:5000/api/v1/predict?ram_gb=8&weight_kg=2,3&inches=15.6

Respuesta (JSON):

{
  "predictions": "Gaming"
}

/api/v1/retrain/

Descripción: Este endpoint permite reentrenar el modelo de machine learning con nuevos datos.

Método: GET (aunque en producción sería más seguro usar POST)

Ejemplo de respuesta:

{
  "message": "Modelo reentrenado con éxito",
  "status": "updated"
}

/status

Descripción: Este endpoint devuelve el estado actual de la API.

Método: GET

Ejemplo de respuesta:

{
  "status": "OK",
  "message": "La API está funcionando correctamente"
}
